# TUI 使用说明

## 启动

使用 Go 1.26+ 构建后，可通过以下命令进入 TUI：

```bash
coding-agent tui
```

## 界面布局

从上到下，界面按固定顺序渲染（长会话下顺序不变）：

1. **标题** — `coding-agent TUI`
2. **消息区** — 仅包含 `user` / `assistant` / `system` 角色消息（自动换行）；无消息时显示 `(暂无消息)`
3. **输入区** — 空闲时 `> ` 加当前草稿；处理中见下文
4. **错误区**（有错误时）— `错误: …`，与消息区分离，不会混入消息正文
5. **状态区**（有状态时）— 如 `处理中...`、`已中断`，同样独立于消息区
6. **快捷键帮助** — 随当前状态切换文案（见下文）

错误与状态行会占用终端高度，消息区视口高度会相应缩小；出现或消失状态/错误时，滚动位置会自动重新钳制，避免内容跳动越界。

## 快捷键

| 键位 | 行为 |
|------|------|
| `Enter` | 发送输入（trim 后非空且非处理中） |
| `Esc` | 中断当前轮输出（不退出会话；显示 `已中断`） |
| `Ctrl+C` | 若正在处理则先中断，然后退出 TUI |
| `Up` / `Down` | 在消息视口内上下滚动（处理中仍可用） |
| `j` / `k` | **仅当输入框为空时** 等同 Down / Up 滚动；输入框有内容时作为普通字符输入 |

### 键位与输入的冲突规则

- **输入框为空**：`j` / `k` 用于滚动，与 `Up` / `Down` 语义一致。
- **输入框非空**：`j` / `k` 追加到草稿，不触发滚动（避免编辑时误滚）。
- **处理中（busy）**：禁止追加字符与退格；输入区显示 `> (处理中，Esc 可中断)`，不展示草稿；`Enter` 无效；`Esc` 可中断；`Up`/`Down` 仍可滚动；输入框为空时 `j`/`k` 仍可滚动。

### 帮助行随状态变化

- **处理中**：`Esc 中断当前轮 · Ctrl+C 退出`
- **有错误或刚中断**（`lastError` 非空或状态为 `已中断`）：`可继续输入并 Enter 发送 · ↑↓/jk 滚动 · Ctrl+C 退出`
- **默认**：`↑↓/jk 滚动 · Enter 发送 · Esc 中断 · Ctrl+C 退出`

## 处理中与流式输出

提交后（`Enter`）：

- 清空输入框，进入 **busy**；显示状态 `处理中...`，输入区提示处理中。
- 追加用户消息与空的 assistant 占位，assistant 内容随流式 chunk 增量更新。
- 流式输出期间，若用户曾上滚，新 chunk 到达时会 **自动滚到底部** 跟随最新内容。
- 轮次正常结束（`StreamDone`）后清除 busy 与状态，恢复可输入。

按 `Esc` 中断当前轮：

- 取消进行中的 turn，清除 busy；状态显示 `已中断`。
- 已输出的 assistant 片段保留；后续因取消产生的 stream 错误 **不会** 写入错误区或改动消息正文。
- 中断后可立即继续输入并发送下一轮。

## 错误与恢复

- 可恢复的运行时错误（如超时、网络）显示在 **错误区**（`错误: …`），不进入消息区。
- 错误出现后界面仍可使用：继续输入并 `Enter` 提交会 **清除错误** 并开始新轮。
- 用户主动中断后的取消类错误不会当作 `lastError` 展示。

## 长会话滚动

- 消息按终端宽度换行后形成多行；视口一次只显示其中一段，由 `scrollOffset` 控制。
- 新会话默认在顶部；内容超出视口后，**流式 chunk 更新**会 **自动滚到底部** 跟随最新内容；**提交新轮**时若用户已上滚则 **保持当前阅读位置**（仅钳制到合法范围，不强制跳底）。
- `Up`/`Down`（及空输入时的 `j`/`k`）逐行滚动，在顶部/底部边界处停止。
- 终端窗口尺寸变化时，滚动位置会被钳制到合法范围；视口变小时不会把偏移“推得更远”。

## 当前能力范围

- 并行于 `chat`/`once` 的新入口，不替换原有命令
- 消息流展示与输入区
- Agent 会话对接与增量流式渲染
- 错误提示可视化与中断反馈
- 布局分区、busy 输入锁、键位冲突规则与长会话滚动（行为见上文，并由 `internal/tui` 测试覆盖）

## 已知限制

- 当前会话环境若低于 Go 1.26，可能无法完成全仓构建验证
- TUI 下需要交互审批的高风险工具调用当前默认拒绝（避免 raw stdin 询问导致阻塞）
- 首版不包含会话管理面板、配置面板、多窗口布局
- 主题与鼠标交互仅提供基础能力

## 后续扩展方向

- 会话列表与恢复的可视化管理
- 任务/子代理状态看板
- 更完整的主题与布局配置
